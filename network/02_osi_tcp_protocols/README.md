## Основные концепции

### Модель OSI (7 уровней)

| Уровень | Название | Протоколы | Утилиты |
|---------|----------|-----------|---------|
| **7** | Прикладной (Application) | HTTP, DNS, FTP | curl, wget |
| **6** | Представления (Presentation) | SSL/TLS, кодировки | — |
| **5** | Сеансовый (Session) | NetBIOS, RPC | — |
| **4** | Транспортный (Transport) | TCP, UDP | tcpdump, netstat |
| **3** | Сетевой (Network) | IP, ICMP | ping, traceroute, tcpdump |
| **2** | Канальный (Data Link) | Ethernet, ARP | ip addr, tcpdump |
| **1** | Физический (Physical) | Кабель, радиоволны | — |

---

### Модель TCP/IP (4 уровня)

| Уровень | Название | Соответствие OSI | Протоколы |
|---------|----------|------------------|-----------|
| **4** | Прикладной | OSI 5-7 | HTTP, DNS, FTP, SMTP |
| **3** | Транспортный | OSI 4 | TCP, UDP |
| **2** | Интернет | OSI 3 | IP, ICMP, ARP |
| **1** | Канальный | OSI 1-2 | Ethernet, Wi-Fi |

## Утилиты и протоколы

### tcpdump
- **Назначение:** Перехват и анализ сетевых пакетов
- **Уровни OSI:** 2-7 (от Ethernet до HTTP)
- **Протоколы:** ICMP, TCP, UDP, HTTP, DNS
- **Применение:** Отладка сетевых проблем, анализ трафика

### ping
- **Назначение:** Проверка доступности узла
- **Уровень OSI:** 3 (сетевой)
- **Протокол:** ICMP (Echo Request/Reply)
- **Применение:** Проверка связности, измерение RTT

### traceroute
- **Назначение:** Определение маршрута до узла
- **Уровень OSI:** 3 (сетевой)
- **Протокол:** ICMP/UDP (с увеличивающимся TTL)
- **Применение:** Поиск узких мест, диагностика маршрутизации

### ip
- **Назначение:** Управление сетевыми интерфейсами и маршрутизацией
- **Уровни OSI:** 2-3 (канальный, сетевой)
- **Подкоманды:** addr, route, link
- **Применение:** Конфигурация сети, просмотр маршрутов

## Практические примеры

### Пример 1: Анализ HTTP-запроса

**Команда:**
```bash
tcpdump -i enp0s3 'tcp port 80' -n -v -A
```

**Результат:** Видны HTTP-заголовки (GET, Host, User-Agent) и ответ сервера (200 OK, Content-Type).

---

### Пример 2: Диагностика медленного соединения

**Шаги:**
1. `ping 8.8.8.8` — проверка RTT (76 мс)
2. `traceroute 8.8.8.8` — поиск узла с высокой задержкой (hop 2: 84 мс)
3. **Вывод:** Задержка появляется на втором хопе (провайдер)

---

### Пример 3: Проверка маршрута

**Команда:**
```bash
ip route get 8.8.8.8
```

**Результат:**
```
8.8.8.8 via 192.168.1.1 dev enp0s3 src 192.168.1.35 uid 0
```

**Интерпретация:** Пакеты до 8.8.8.8 идут через шлюз 192.168.1.1 на интерфейсе enp0s3.
