**Анализ маршрута:**
- **Hop 1:** `192.168.1.1` — локальный шлюз (роутер), задержка ~1 мс

## Сравнение утилит по уровням OSI

| Утилита | Уровень OSI | Протокол | Назначение |
|---------|-------------|----------|------------|
| **ping** | 3 (Network) | ICMP | Проверка доступности узла, измерение RTT |
| **traceroute** | 3 (Network) | ICMP/UDP | Определение маршрута пакетов, поиск узких мест |
| **ip addr** | 2-3 (Data Link, Network) | — | Просмотр IP/MAC-адресов, конфигурации интерфейсов |
| **ip route** | 3 (Network) | — | Просмотр таблицы маршрутизации |

---

### Как работают утилиты на разных уровнях

#### ping (уровень 3)
- **Протокол:** ICMP (часть IP-стека)
- **Работа:** Отправляет ICMP Echo Request, ждёт ICMP Echo Reply
- **Что даёт:** Доступность узла, время отклика (RTT), потери пакетов
- **Уровень OSI:** 3 (сетевой) — работает с IP-адресами

#### traceroute (уровень 3)
- **Протокол:** ICMP/UDP (отправляет пакеты с увеличивающимся TTL)
- **Работа:** Каждый узел уменьшает TTL на 1, при TTL=0 отправляет ICMP Time Exceeded
- **Что даёт:** Список промежуточных узлов, время до каждого хопа
- **Уровень OSI:** 3 (сетевой) — показывает маршрутизацию IP-пакетов

#### ip addr (уровни 2-3)
- **Работа:** Читает конфигурацию из ядра Linux
- **Что даёт:** IP-адреса (уровень 3), MAC-адреса (уровень 2), состояние интерфейсов
- **Уровень OSI:** 2 (канальный) и 3 (сетевой)

#### ip route (уровень 3)
- **Работа:** Показывает таблицу маршрутизации ядра
- **Что даёт:** Как ядро принимает решение о направлении пакетов
- **Уровень OSI:** 3 (сетевой) — управление маршрутизацией IP

## Практический анализ

### Почему ping и traceroute дают разное время?

**ping до 8.8.8.8:**
```
rtt min/avg/max/mdev = 72.876/76.769/85.903/4.823 ms
```

**traceroute до 8.8.8.8 (последний хоп):**
```
9  142.250.229.84  102.371 ms * 8.8.8.8  87.136 ms
```

**Причина разницы:**
- **ping** измеряет время туда-обратно (RTT) напрямую
- **traceroute** отправляет пакеты с разными TTL, проходя через промежуточные узлы
- **traceroute** может использовать другой путь или протокол (UDP вместо ICMP)
- **ICMP rate limiting** на промежуточных узлах может замедлять ответы

## Диагностика проблем с сетью

### Пример 1: Узел недоступен

Если `ping` не получает ответов:
1. Проверить `ip addr` — есть ли IP-адрес на интерфейсе?
2. Проверить `ip route` — есть ли маршрут до узла?
3. Запустить `traceroute` — где обрывается маршрут?

### Пример 2: Медленное соединение

Если `ping` показывает высокий RTT:
1. Проверить `ip -s link` — есть ли dropped packets?
2. Запустить `traceroute` — на каком хопе появляется задержка?
3. Проверить `ping` до промежуточных узлов
