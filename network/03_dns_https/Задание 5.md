## Ключевые различия

### HTTP/1.1

**Механизм работы:**
```
Клиент --------[GET /style1.css]-------> Сервер
Клиент <-------[200 OK + данные]-------- Сервер
Клиент --------[GET /style2.css]-------> Сервер  ← ждёт завершения предыдущего
Клиент <-------[200 OK + данные]-------- Сервер
```

**Ограничения:**
- Одно TCP-соединение → один запрос
- Head-of-line blocking (очередь запросов)
- Браузер открывает 6-8 соединений параллельно
- Текстовый протокол → большие заголовки
- Повторяющиеся заголовки в каждом запросе

---

### HTTP/2

**Механизм работы:**
```
                [Stream 1: GET /style1.css]
Клиент --------[Stream 2: GET /style2.css]-------> Сервер
                [Stream 3: GET /style3.css]
                
                [Stream 1: 200 + данные]
Клиент <-------[Stream 2: 200 + данные]------------ Сервер
                [Stream 3: 200 + данные]
```

**Преимущества:**
- **Мультиплексирование:** много запросов в одном соединении
- **Нет head-of-line blocking:** запросы обрабатываются параллельно
- **Одно TCP-соединение:** экономия на handshake
- **Бинарный протокол:** эффективнее парсинг
- **HPACK сжатие заголовков:** меньше трафика
- **Приоритизация потоков:** важные ресурсы первыми
- **Server Push:** сервер может отправить ресурсы до запроса

## Визуальное сравнение

### HTTP/1.1 - Timeline (упрощённо)
```
Соединение 1: [HTML........]
Соединение 2: [CSS1....]
Соединение 3: [CSS2....]
Соединение 4: [CSS3....]
Соединение 5: [JS1.....]
Соединение 6: [JS2.....]
Соединение 7 (ждёт): ...[JS3.....]

Итого: 7 TCP-соединений, последовательная обработка
```

### HTTP/2 - Timeline
```
Одно TCP-соединение:
Stream 1: [HTML........]
Stream 2: [CSS1....]
Stream 3: [CSS2....]
Stream 4: [CSS3....]
Stream 5: [JS1.....]
Stream 6: [JS2.....]
Stream 7: [JS3.....]

Итого: 1 TCP-соединение, параллельная обработка
```
