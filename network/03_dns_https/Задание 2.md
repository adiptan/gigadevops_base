# Задание 2: Построение локального DNS-сервера

**Статус:** ✅ Выполнено  
**Дата:** 2026-03-08  
**Платформа:** Linux (Ubuntu)

---

## Описание задания

Установить и настроить локальный DNS-сервер BIND9:
- Установить BIND
- Настроить локальную зону (local.test)
- Создать записи: A, CNAME, MX
- Проверить работу зоны

**Конечный результат:** Локальный домен (www.local.test) разрешается в указанный IP-адрес.

---

### Пункт 2: Настройка локальной зоны

#### 2.1. Добавление зоны в конфигурацию

**Файл:** `/etc/bind/named.conf.local`

**Содержимое:**
```bind
zone "local.test" {
    type master;
    file "/etc/bind/zones/db.local.test";
};
```

✅ Зона local.test добавлена в конфигурацию

---

#### 2.2. Создание файла зоны

**Файл:** `/etc/bind/zones/db.local.test`

**Содержимое:**
```bind
$TTL    604800
@       IN      SOA     ns1.local.test. admin.local.test. (
                              3         ; Serial
                         604800         ; Refresh
                          86400         ; Retry
                        2419200         ; Expire
                         604800 )       ; Negative Cache TTL
;
@       IN      NS      ns1.local.test.
@       IN      A       192.168.1.1
ns1     IN      A       192.168.1.10
www     IN      A       192.168.1.20
mail    IN      A       192.168.1.30
ftp     IN      CNAME   www
blog    IN      A       192.168.1.40
@       IN      MX      10 mail.local.test.
```

**Записи:**
- **SOA** - Start of Authority (ns1.local.test)
- **NS** - Name Server (ns1.local.test)
- **A-записи:**
  - @ (local.test) → 192.168.1.1
  - ns1.local.test → 192.168.1.10
  - www.local.test → 192.168.1.20
  - mail.local.test → 192.168.1.30
  - blog.local.test → 192.168.1.40
- **CNAME:**
  - ftp.local.test → www.local.test
- **MX:**
  - local.test → mail.local.test (приоритет 10)

✅ Файл зоны создан со всеми типами записей

### Пункт 5: Проверка работы зоны

#### 5.1. A-запись (www.local.test)

**Команда:**
```bash
dig @localhost www.local.test
```

**Вывод:**
```
; <<>> DiG 9.18.39-0ubuntu0.24.04.2-Ubuntu <<>> @localhost www.local.test
;; ANSWER SECTION:
www.local.test.		604800	IN	A	192.168.1.20

;; Query time: 0 msec
;; SERVER: 127.0.0.1#53(localhost) (UDP)
```

**Результат:** www.local.test → **192.168.1.20**

✅ A-запись работает

---

#### 5.2. CNAME-запись (ftp.local.test)

**Команда:**
```bash
dig @localhost ftp.local.test
```

**Вывод:**
```
;; ANSWER SECTION:
ftp.local.test.		604800	IN	CNAME	www.local.test.
www.local.test.		604800	IN	A	192.168.1.20
```

**Результат:**
- ftp.local.test → CNAME → www.local.test
- www.local.test → **192.168.1.20**

✅ CNAME-запись работает (алиас корректно разрешается)

---

#### 5.3. MX-запись (local.test)

**Команда:**
```bash
dig @localhost local.test MX
```

**Вывод:**
```
;; ANSWER SECTION:
local.test.		604800	IN	MX	10 mail.local.test.

;; ADDITIONAL SECTION:
mail.local.test.	604800	IN	A	192.168.1.30
```

**Результат:**
- Почтовый сервер: **mail.local.test** (приоритет 10)
- IP почтового сервера: **192.168.1.30**

✅ MX-запись работает
