# Задание 1: Настройка и анализ DNS-запросов

**Статус:** ✅ Выполнено  
**Дата:** 2026-03-08  
**Платформа:** Linux (Ubuntu)

---

## Описание задания

Установить утилиты для работы с DNS (dnsutils) и выполнить анализ различных типов DNS-записей:
- Определить IP-адрес для домена
- Найти серверы имен (NS-записи)
- Проверить почтовые серверы (MX-записи)
- Определить обратное разрешение (PTR-запись)

**Конечный результат:** Получены корректные данные для всех типов записей.

---

## Выполнение

### Пункт 1: Установка dnsutils

**Команда:**
```bash
apt-get update && apt-get install -y dnsutils
```

**Результат:**
```
dnsutils (1:9.18.39-0ubuntu0.24.04.2) установлен
```

✅ Пакет dnsutils установлен, включает утилиты: dig, nslookup, host

---

### Пункт 2: Определение IP-адреса для домена (A-запись)

**Команда:**
```bash
dig www.example.com
```

**Вывод:**
```
; <<>> DiG 9.18.39-0ubuntu0.24.04.2-Ubuntu <<>> www.example.com
;; global options: +cmd
;; Got answer:
;; ->>HEADER<<- opcode: QUERY, status: NOERROR, id: 7873
;; flags: qr rd ra; QUERY: 1, ANSWER: 2, AUTHORITY: 0, ADDITIONAL: 1

;; QUESTION SECTION:
;www.example.com.		IN	A

;; ANSWER SECTION:
www.example.com.	249	IN	A	8.47.69.8
www.example.com.	249	IN	A	8.6.112.8

;; Query time: 0 msec
;; SERVER: 127.0.0.53#53(127.0.0.53) (UDP)
;; WHEN: Sun Mar 08 08:33:11 MSK 2026
;; MSG SIZE  rcvd: 76
```

**Результат:**
- www.example.com → **8.47.69.8** и **8.6.112.8**
- Получено 2 IP-адреса (load balancing/failover)

✅ A-записи получены успешно

---

### Пункт 3: Поиск серверов имен (NS-записи)

**Команда:**
```bash
dig example.com NS
```

**Вывод:**
```
; <<>> DiG 9.18.39-0ubuntu0.24.04.2-Ubuntu <<>> example.com NS
;; ANSWER SECTION:
example.com.		71820	IN	NS	hera.ns.cloudflare.com.
example.com.		71820	IN	NS	elliott.ns.cloudflare.com.

;; Query time: 80 msec
;; SERVER: 127.0.0.53#53(127.0.0.53) (UDP)
```

**Результат:**
- NS-серверы example.com:
  - **hera.ns.cloudflare.com**
  - **elliott.ns.cloudflare.com**

✅ NS-записи получены (Cloudflare DNS)

---

### Пункт 4: Проверка почтовых серверов (MX-записи)

**Команда:**
```bash
dig google.com MX
```

**Вывод:**
```
; <<>> DiG 9.18.39-0ubuntu0.24.04.2-Ubuntu <<>> google.com MX
;; ANSWER SECTION:
google.com.		235	IN	MX	10 smtp.google.com.

;; Query time: 80 msec
;; SERVER: 127.0.0.53#53(127.0.0.53) (UDP)
```

**Результат:**
- Почтовый сервер google.com: **smtp.google.com** (приоритет 10)

✅ MX-запись получена

*Примечание: example.com не имеет реальных MX-записей (нулевая запись), поэтому использован google.com для демонстрации.*

---

### Пункт 5: Обратное разрешение (PTR-запись)

**Команда:**
```bash
dig -x 8.8.8.8
```

**Вывод:**
```
; <<>> DiG 9.18.39-0ubuntu0.24.04.2-Ubuntu <<>> -x 8.8.8.8
;; QUESTION SECTION:
;8.8.8.8.in-addr.arpa.		IN	PTR

;; ANSWER SECTION:
8.8.8.8.in-addr.arpa.	71984	IN	PTR	dns.google.

;; Query time: 76 msec
;; SERVER: 127.0.0.53#53(127.0.0.53) (UDP)
```

**Результат:**
- IP 8.8.8.8 → **dns.google** (Google Public DNS)

✅ PTR-запись получена (обратное разрешение работает)

---

### Дополнительно: nslookup

**Команда:**
```bash
nslookup www.example.com
```

**Вывод:**
```
Server:		127.0.0.53
Address:	127.0.0.53#53

Non-authoritative answer:
Name:	www.example.com
Address: 8.6.112.8
Address: 8.47.69.8
Address: 2606:4700::6812:1a78
Address: 2606:4700::6812:1b78
```

**Результат:**
- Получены IPv4 и IPv6 адреса
- Работает через локальный резолвер systemd-resolved

✅ nslookup работает корректно
