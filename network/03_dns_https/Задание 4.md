## Описание задания

Создать самоподписанный SSL-сертификат и настроить веб-сервер для работы по HTTPS:
- Установить OpenSSL

### Пункт 2: Генерация самоподписанного SSL-сертификата

#### 2.1. Создание директорий

**Команды:**
```bash
mkdir -p /etc/ssl/private /etc/ssl/certs
```

✅ Директории созданы

---

#### 2.2. Генерация ключа и сертификата

**Команда:**
```bash
openssl req -x509 -nodes -days 365 -newkey rsa:2048 \
  -keyout /etc/ssl/private/local.test.key \
  -out /etc/ssl/certs/local.test.crt \
  -subj "/C=RU/ST=Krasnodar/L=Krasnodar/O=GigaDevOps/OU=IT/CN=local.test/emailAddress=admin@local.test"
```

**Параметры:**
- `-x509` - создать самоподписанный сертификат
- `-nodes` - не шифровать приватный ключ
- `-days 365` - срок действия 365 дней
- `-newkey rsa:2048` - создать RSA ключ 2048 бит
- `-keyout` - путь к приватному ключу
- `-out` - путь к сертификату
- `-subj` - информация о сертификате

**Результат:**
```
Generating a RSA private key
..........++++++++...++++++++
writing new private key to '/etc/ssl/private/local.test.key'
-----
```

✅ Сертификат и ключ созданы

---

### Пункт 3: Настройка Nginx для HTTPS

#### 3.1. Проверка Nginx

**Команда:**
```bash

```

**Результат:**
```

```

✅ Nginx установлен

---

#### 3.2. Создание конфигурации сайта

**Файл:** `/etc/nginx/sites-available/local.test`

**Содержимое:**
```nginx
server {
    listen 443 ssl http2;
    listen [::]:443 ssl http2;
    server_name local.test www.local.test;

    ssl_certificate /etc/ssl/certs/local.test.crt;
    ssl_certificate_key /etc/ssl/private/local.test.key;

    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
    ssl_prefer_server_ciphers on;

    root /var/www/local.test;
    index index.html;

    location / {
        try_files $uri $uri/ =404;
    }
}

server {
    listen 80;
    listen [::]:80;
    server_name local.test www.local.test;
    return 301 https://$server_name$request_uri;
}
```

**Особенности конфигурации:**
- HTTPS на порту 443 с HTTP/2
- TLS 1.2 и 1.3 (современные безопасные протоколы)
- Редирект с HTTP → HTTPS
- Поддержка IPv4 и IPv6

✅ Конфигурация создана

#### 4.2. Проверка информации о сертификате
