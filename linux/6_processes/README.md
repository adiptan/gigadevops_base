## 🎯 Изученные команды и утилиты

### Просмотр процессов:
- `ps -aux` - список всех процессов
- `ps --sort=-%cpu` - сортировка по CPU
- `ps --sort=-%mem` - сортировка по памяти
- `pstree -p` - дерево процессов с PID

### Управление процессами:
- `command &` - запуск в фоновом режиме
- `jobs -l` - список фоновых задач
- `fg %1` - перевод на передний план
- `bg %1` - возврат в фоновый режим
- `Ctrl+Z` - приостановка процесса
- `kill -TERM PID` - корректное завершение
- `kill -9 PID` - принудительное завершение

### Приоритеты:
- `nice -n 10 command` - запуск с приоритетом
- `renice +10 -p PID` - изменение приоритета
- Шкала: `-20` (высший) до `+19` (низший)

### Мониторинг:
- `top` - мониторинг в реальном времени
- `htop` - интерактивный мониторинг
- `atop -w file` - запись логов
- `atop -r file` - чтение логов

### Файлы процессов:
- `/proc/[PID]/stat` - статистика процесса
- `/proc/[PID]/fd/` - файловые дескрипторы
- `/proc/meminfo` - информация о памяти
- `lsof -p PID` - открытые файлы процесса

## 📊 Ключевые концепции

### 1. Процессы и их состояния
- **R** (Running) - выполняется
- **S** (Sleeping) - ожидание события
- **D** (Disk sleep) - ожидание I/O
- **Z** (Zombie) - завершён, ждёт родителя
- **T** (Traced/Stopped) - остановлен или трассируется

### 2. Приоритеты (Nice)
- Значения от `-20` (максимальный приоритет) до `+19` (минимальный)
- `0` - нормальный приоритет
- Только root может понижать nice (увеличивать приоритет)

### 3. Файловые дескрипторы
- **0** - stdin (стандартный ввод)
- **1** - stdout (стандартный вывод)
- **2** - stderr (ошибки)
- Дополнительные FD для открытых файлов

### 4. Сигналы
- **TERM (15)** - корректное завершение (по умолчанию)
- **KILL (9)** - принудительное завершение
- **STOP (19)** - приостановка
- **CONT (18)** - возобновление

### 5. Утилиты мониторинга

**top:**
- Реальное время
- Базовая функциональность
- Встроен во все дистрибутивы

**htop:**
- Интерактивный интерфейс
- Цветовая индикация
- Управление клавишами F1-F10

**atop:**
- Историческая статистика
- Детальная информация по ресурсам
- Логирование для ретроспективного анализа

## 💡 Практические выводы

1. **ps и pstree** - базовые инструменты для просмотра процессов
2. **top/htop** - для интерактивного мониторинга
3. **atop** - для анализа истории и поиска проблем
4. **/proc** - богатый источник информации о процессах
5. **nice/renice** - управление приоритетами
6. **kill** - корректное завершение процессов
