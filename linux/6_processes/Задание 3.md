## Пункт 2 - Мониторинг с помощью htop

### Пункт 2.1 - Установка и запуск htop

**Проверка установки:**
```bash
which htop
```

**Вывод:**
```
/usr/bin/htop
```

**Результат:** htop установлен и готов к использованию

---

### Пункт 2.2 - Поиск фонового процесса в htop

**Интерфейс htop показывает:**
- **Древовидная структура** процессов (F5)
- **Цветовая индикация** загрузки CPU
- **Фильтрация** по имени процесса (F4)

**Для поиска фонового yes:**
- Нажать **F4** (Filter)
- Ввести **"yes"**
- Процесс подсветится в списке

---

### Пункт 2.3 - Изменение приоритета (Nice) в htop

**Шаги:**
1. Выбрать процесс стрелками
2. Нажать **F7** (Nice -) для увеличения приоритета
3. Или **F8** (Nice +) для уменьшения приоритета
4. Изменения применяются мгновенно

**Пример:**
- Процесс `yes` с Nice 0
- После нажатия F8 (5 раз) → Nice 5
- Приоритет понижен

**Преимущества htop:**
- Интерактивное управление (без команд)
- Визуализация загрузки CPU/памяти
- Цветовая индикация
- Дерево процессов

## Пункт 3 - Изучение истории с помощью atop

### Пункт 3.1 - Установка atop

**Проверка установки:**
```bash
which atop
```

**Вывод:**
```
/usr/bin/atop
```

**Результат:** atop установлен

---

### Пункт 3.2 - Запуск atop с записью логов

**Команда:**
```bash
atop -w /tmp/atop.log 1 3
```

**Параметры:**
- `-w /tmp/atop.log` - запись в файл
- `1` - интервал 1 секунда
- `3` - количество снимков

**Результат:** Лог создан в `/tmp/atop.log`
