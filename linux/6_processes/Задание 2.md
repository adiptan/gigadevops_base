### Пункт 1.3 - Управление процессом (fg/bg)

**Команды:**
```bash
# Запуск в фоне
ping google.com > /tmp/output.log 2>&1 &

# Просмотр фоновых задач
jobs -l

# Перевод на передний план (для демонстрации)
# fg %1

# Приостановка (Ctrl+Z) и возврат в фон
# bg %1
```

**Демонстрация jobs -l:**
```
[1]+ 86198 Running                 ping google.com > /tmp/output.log 2>&1 &
```

**Примечание:** Команды `fg` и `bg` работают в интерактивном shell. В скриптах используется `&` для фонового запуска.

## Пункт 2 - Установка приоритетов процессам

### Пункт 2.1 - Запуск процесса yes

**Команда:**
```bash
yes > /dev/null &
```

**Вывод:**
```
[1] 86178
```

**Проверка процесса:**
```bash
ps -p 86178 -o pid,comm,nice,%cpu
```

**Вывод:**
```
    PID COMMAND          NI %CPU
  86178 yes               0 99.9
```

**Результат:**
- PID: 86178
- Nice: 0 (нормальный приоритет)
- CPU: 99.9% (нагружает процессор)

---

### Пункт 2.2 - Изменение приоритета с помощью renice

**Команда:**
```bash
sudo renice +10 -p 86178
```

**Вывод:**
```
86178 (process ID) old priority 0, new priority 10
```

**Проверка после изменения:**
```bash
ps -p 86178 -o pid,comm,nice,%cpu
```

**Вывод:**
```
    PID COMMAND          NI %CPU
  86178 yes              10  100
```

**Результат:**
- Nice изменён с 0 на +10 (более низкий приоритет)
- Процесс всё ещё нагружает CPU, но с меньшим приоритетом планировщика

**Шкала приоритетов:**
- `-20` до `+19` (чем меньше, тем выше приоритет)
- `0` - нормальный приоритет
- `+10` - пониженный приоритет

### Пункт 3.2 - Завершение процесса с помощью kill

**Команда:**
```bash
kill -TERM 86178
```

**Проверка завершения:**
```bash
ps -p 86178
```

**Вывод:**
```
    PID TTY          TIME CMD
(процесс не найден)
```

**Результат:** Процесс успешно завершён сигналом TERM

**Типы сигналов kill:**
- `TERM` (15) - корректное завершение (по умолчанию)
- `KILL` (9) - принудительное завершение
- `STOP` (19) - приостановка процесса
- `CONT` (18) - возобновление процесса
