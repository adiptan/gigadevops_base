## Требования
1. Просмотр информации о месте с помощью df -h

3. Использование ncdu для анализа
4. Проверка SMART-дисков

## Пункт 1: Просмотр информации о дисковом пространстве

### Анализ свободного и занятого пространства:
```bash
df -h | grep -E "(Filesystem|/mnt/disk)"
```

### Вывод (до создания файлов):
```
Filesystem      Size  Used Avail Use% Mounted on
/dev/loop4p1    989M   24K  922M   1% /mnt/disk1
/dev/loop16p1   447M   35M  413M   8% /mnt/disk2
/dev/loop17p1   988M   24K  921M   1% /mnt/disk3
```

**Анализ:**
- disk1 (ext4): 989M общий размер, использовано 24K, доступно 922M
- disk2 (xfs): 447M общий размер, использовано 35M (метаданные FS), доступно 413M
- disk3 (ext4): 988M общий размер, использовано 24K, доступно 921M

### Создание крупных тестовых файлов:

#### Файл 500 МБ на disk1:
```bash
dd if=/dev/zero of=/mnt/disk1/file_500M bs=1M count=500
```

Вывод:
```
500+0 records in
500+0 records out
524288000 bytes (524 MB, 500 MiB) copied, 0.236118 s, 2.2 GB/s
```

#### Файл 400 МБ на disk2:
```bash
dd if=/dev/zero of=/mnt/disk2/file_400M bs=1M count=400
```

Вывод:
```
400+0 records in
400+0 records out
419430400 bytes (419 MB, 400 MiB) copied, 0.0922997 s, 4.5 GB/s
```

#### Файл 700 МБ на disk3:
```bash
dd if=/dev/zero of=/mnt/disk3/file_700M bs=1M count=700
```

Вывод:
```
700+0 records in
700+0 records out
734003200 bytes (734 MB, 700 MiB) copied, 0.355069 s, 2.1 GB/s
```

### Проверка использования после создания файлов:
```bash
df -h | grep "/mnt/disk"
```

### Вывод:
```
/dev/loop4p1    989M  501M  422M  55% /mnt/disk1
/dev/loop16p1   447M  435M   13M  98% /mnt/disk2
/dev/loop17p1   988M  701M  221M  77% /mnt/disk3
```

**Результат:**
- disk1: использовано 55% (501M из 989M)
- disk2: использовано 98% (435M из 447M) - почти заполнен!
- disk3: использовано 77% (701M из 988M)

## Пункт 2: Поиск больших файлов с помощью du

### Анализ всех дисков с сортировкой:
```bash
du -h /mnt/disk* --max-depth=1 | sort -h
```

### Вывод:
```
16K	/mnt/disk1/lost+found
16K	/mnt/disk3/lost+found
400M	/mnt/disk2
501M	/mnt/disk1
701M	/mnt/disk3
```

### Поиск файлов больше 100 МБ:
```bash
find /mnt/disk* -type f -size +100M -exec du -h {} \;
```

### Вывод:
```
501M	/mnt/disk1/file_500M
400M	/mnt/disk2/file_400M
701M	/mnt/disk3/file_700M
```

**Найдено 3 больших файла:**
1. file_500M на disk1 (501 МБ)
2. file_400M на disk2 (400 МБ)
3. file_700M на disk3 (701 МБ)

### Детальный анализ disk1:
```bash
du -sh /mnt/disk1/*
```

### Вывод:
```
501M	/mnt/disk1/file_500M
16K	/mnt/disk1/lost+found
```

## Пункт 3: Использование ncdu для анализа

### Установка ncdu:
```bash
apt-get install -y ncdu
```

### Вывод:
```
Setting up ncdu (1.19-0.1) ...
```

### Анализ диска с помощью ncdu:
```bash
ncdu -0 -o /tmp/ncdu_disk1.json /mnt/disk1
```

**ncdu (NCurses Disk Usage)** - это интерактивный анализатор использования дискового пространства с удобным текстовым интерфейсом.

Опции команды:
- `-0` - не обновлять экран во время сканирования
- `-o` - экспорт результатов в JSON
- `/mnt/disk1` - сканируемая директория

### Просмотр содержимого через du:
```bash
du -sh /mnt/disk1/*
```

### Результаты анализа ncdu:
```
501M	/mnt/disk1/file_500M
16K	/mnt/disk1/lost+found
```

**Преимущества ncdu:**
- Интерактивная навигация по директориям
- Сортировка по размеру
- Быстрое удаление файлов прямо из интерфейса
- Визуализация использования пространства
- Экспорт/импорт результатов сканирования

## Пункт 4: Проверка состояния SMART-дисков

### Установка smartmontools:
```bash
apt-get install -y smartmontools
```

### Вывод:
```
Setting up smartmontools (7.4-2build1) ...
```

### Проверка SMART для loop-устройств:
```bash
smartctl -i /dev/loop4
```

### Вывод:
```
smartctl 7.4 2023-08-01 r5530 [x86_64-linux-6.17.0-14-generic] (local build)
Copyright (C) 2002-23, Bruce Allen, Christian Franke, www.smartmontools.org

/dev/loop4: Unable to detect device type
Please specify device type with the -d option.

Use smartctl -h to get a usage summary
```

**Примечание:** Loop-устройства (виртуальные диски из файлов) не поддерживают SMART, так как это технология мониторинга физических жёстких дисков.

### Проверка SMART физического диска:
```bash
smartctl -i /dev/sda
```

### Вывод:
```
smartctl 7.4 2023-08-01 r5530 [x86_64-linux-6.17.0-14-generic] (local build)
Copyright (C) 2002-23, Bruce Allen, Christian Franke, www.smartmontools.org

=== START OF INFORMATION SECTION ===
Device Model:     VBOX HARDDISK
Serial Number:    VB8337b055-babdc34d
Firmware Version: 1.0
User Capacity:    54,741,434,368 bytes [54.7 GB]
Sector Size:      512 bytes logical/physical
Device is:        Not in smartctl database 7.3/5528
ATA Version is:   ATA/ATAPI-6 published, ANSI INCITS 361-2002
Local Time is:    Sat Feb 28 10:21:31 2026 MSK
SMART support is: Unavailable - device lacks SMART capability.
```

**Примечание:** В данном случае система работает в VirtualBox, где диски также не поддерживают SMART.

### Пример использования smartctl на реальном оборудовании:

На физическом сервере команды для полной проверки SMART:

```bash
# Получение SMART информации
smartctl -i /dev/sda

# Общее здоровье диска
smartctl -H /dev/sda

# Детальная информация обо всех атрибутах
smartctl -A /dev/sda

# Запуск короткого теста
smartctl -t short /dev/sda

# Запуск полного теста
smartctl -t long /dev/sda

# Просмотр результатов теста
smartctl -l selftest /dev/sda
```

**Ключевые SMART атрибуты для мониторинга:**
- Reallocated_Sector_Ct - количество переназначенных секторов
- Current_Pending_Sector - секторы, ожидающие переназначения
- Offline_Uncorrectable - неисправимые секторы
- Temperature_Celsius - температура диска
- Power_On_Hours - время работы диска
