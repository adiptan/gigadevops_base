# Задание 4: Опции монтирования и квоты

## Требования
1. Настроить разные точки монтирования с опциями:
   - Только для чтения (ro)
   - Без кеширования (sync)
   - Ограничение выполнения файлов (noexec)
2. Настройка квот на одном из дисков
3. Создание пользователей и назначение квот

---

## Пункт 1.1: Монтирование в режиме "только для чтения" (ro)

### Размонтирование и монтирование с опцией ro:
```bash
umount /mnt/disk1
mount -o ro /dev/loop4p1 /mnt/disk1
mount | grep "/mnt/disk1"
```

### Вывод:
```
/dev/loop4p1 on /mnt/disk1 type ext4 (ro,relatime)
```

**Опция ro (read-only):**
- Диск доступен только для чтения
- Невозможно создавать, изменять или удалять файлы
- Используется для защиты данных от изменений

### Проверка запрета записи:
```bash
touch /mnt/disk1/test.txt
```

### Вывод:
```
touch: cannot touch '/mnt/disk1/test.txt': Read-only file system
Запись запрещена (ожидаемо)
```

**Результат:** Попытка создания файла заблокирована, так как диск смонтирован в режиме ro.

## Пункт 1.2: Монтирование без кеширования (sync)

### Размонтирование и монтирование с опцией sync:
```bash
umount /mnt/disk1
mount -o sync /dev/loop4p1 /mnt/disk1
mount | grep "/mnt/disk1"
```

### Вывод:
```
/dev/loop4p1 on /mnt/disk1 type ext4 (rw,relatime,sync)
```

**Опция sync:**
- Все операции записи выполняются синхронно
- Данные сразу записываются на диск, минуя кеш
- Снижает производительность, но повышает надёжность
- Полезно для критичных данных (логи, транзакции)

### Тест производительности записи:
```bash
time dd if=/dev/zero of=/mnt/disk1/test_sync bs=1M count=10
```

### Вывод:
```
10485760 bytes (10 MB, 10 MiB) copied, 0.0232344 s, 451 MB/s

real	0m0.043s
user	0m0.002s
sys	0m0.003s
```

**Примечание:** На loop-устройствах sync работает, но разница с async режимом может быть незаметна из-за виртуализации. На реальных дисках sync значительно медленнее.

## Пункт 1.3: Ограничение выполнения файлов (noexec)

### Размонтирование и монтирование с опцией noexec:
```bash
umount /mnt/disk2
mount -o noexec /dev/loop16p1 /mnt/disk2
mount | grep "/mnt/disk2"
```

### Вывод:
```
/dev/loop16p1 on /mnt/disk2 type xfs (rw,noexec,relatime,attr2,inode64,logbufs=8,logbsize=32k,noquota)
```

**Опция noexec:**
- Запрещает выполнение бинарных файлов и скриптов с диска
- Повышает безопасность (нельзя запустить вредоносный код)
- Используется для директорий с данными пользователей

### Создание тестового скрипта:
```bash
echo -e '#!/bin/bash\necho "Hello"' > /mnt/disk2/test.sh
chmod +x /mnt/disk2/test.sh
```

### Попытка выполнения:
```bash
/mnt/disk2/test.sh
```

### Вывод:
```
/usr/bin/bash: line 1: /mnt/disk2/test.sh: Permission denied
Выполнение запрещено (ожидаемо)
```

**Результат:** Скрипт создан с правами на выполнение, но опция noexec блокирует его запуск.

**Применение в реальных системах:**
- `/tmp` - часто монтируется с noexec для защиты от автоматически скачиваемых скриптов
- `/home` - может быть с noexec если пользователи не должны запускать программы
- Директории загрузок веб-серверов

## Пункт 2: Настройка квот

### Установка пакета quota:
```bash
apt-get install -y quota
```

### Вывод:
```
Setting up quota (4.06-1build6) ...
```

### Размонтирование и монтирование с поддержкой квот:
```bash
umount /mnt/disk3
mount -o usrquota,grpquota /dev/loop17p1 /mnt/disk3
mount | grep "/mnt/disk3"
```

### Вывод:
```
/dev/loop17p1 on /mnt/disk3 type ext4 (rw,relatime,quota,usrquota,grpquota)
```

**Опции квот:**
- `usrquota` - включение пользовательских квот
- `grpquota` - включение групповых квот

### Инициализация базы данных квот:
```bash
quotacheck -cug /mnt/disk3
```

**Опции quotacheck:**
- `-c` - создать новые файлы квот
- `-u` - проверить пользовательские квоты
- `-g` - проверить групповые квоты

### Включение квот:
```bash
quotaon /mnt/disk3
```

### Проверка статуса квот:
```bash
repquota -s /mnt/disk3
```

### Вывод:
```
*** Report for user quotas on device /dev/loop17p1
Block grace time: 7days; Inode grace time: 7days
                        Space limits                File limits
User            used    soft    hard  grace    used  soft  hard  grace
----------------------------------------------------------------------
root      --    701M      0K      0K              3     0     0
```

**Пояснение полей:**
- **used** - текущее использование (блоки и inodes)
- **soft** - мягкий лимит (предупреждение)
- **hard** - жёсткий лимит (блокировка)
- **grace** - период отсрочки после превышения soft лимита

## Пункт 3: Создание пользователей и назначение квот

### Создание тестовых пользователей:
```bash
useradd -m testuser1
useradd -m testuser2
```

### Вывод:
```
Пользователи созданы
```

### Установка квот для testuser1:
```bash
setquota -u testuser1 50M 100M 100 200 /mnt/disk3
```

**Параметры:**
- `-u testuser1` - пользователь
- `50M` - soft limit по блокам (50 МБ)
- `100M` - hard limit по блокам (100 МБ)
- `100` - soft limit по inodes (100 файлов)
- `200` - hard limit по inodes (200 файлов)
- `/mnt/disk3` - файловая система

### Вывод:
```
Квота для testuser1 установлена
```

### Установка квот для testuser2:
```bash
setquota -u testuser2 100M 200M 200 400 /mnt/disk3
```

### Вывод:
```
Квота для testuser2 установлена
```

### Проверка установленных квот:
```bash
repquota -s /mnt/disk3 | grep -E "(User|testuser|root)"
```

### Вывод:
```
User            used    soft    hard  grace    used  soft  hard  grace
root      --    701M      0K      0K              3     0     0
```

### Проверка квоты конкретного пользователя:
```bash
quota -u testuser1
```

### Вывод:
```
Disk quotas for user testuser1 (uid 1001): no limited resources used
```

**Это означает:** testuser1 ещё не использует место на /mnt/disk3, но его квоты активны:
- Soft limit: 50 МБ (предупреждение)
- Hard limit: 100 МБ (блокировка)
- Soft limit файлов: 100
- Hard limit файлов: 200

### Полезные команды для работы с квотами:

```bash
# Просмотр квот пользователя
quota -u username

# Просмотр квот группы
quota -g groupname

# Редактирование квот в интерактивном режиме
edquota -u username

# Копирование квот с одного пользователя на другой
edquota -p sourceuser -u targetuser

# Отчёт по всем квотам
repquota -a

# Выключение квот
quotaoff /mnt/disk3
```
