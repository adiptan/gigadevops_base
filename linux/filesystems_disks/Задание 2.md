# Задание 2: Монтирование дисков

## Требования
1. Ручное монтирование разделов
2. Проверка доступности через lsblk и df -h
3. Автоматическое монтирование через /etc/fstab
4. Проверка автоматического монтирования

---

## Пункт 1: Ручное монтирование

### Создание директорий для точек монтирования:
```bash
mkdir -p /mnt/disk1 /mnt/disk2 /mnt/disk3
ls -ld /mnt/disk*
```

### Вывод:
```
drwxr-xr-x 2 root root 4096 Feb 28 10:19 /mnt/disk1
drwxr-xr-x 2 root root 4096 Feb 28 10:19 /mnt/disk2
drwxr-xr-x 2 root root 4096 Feb 28 10:19 /mnt/disk3
```

### Монтирование разделов:
```bash
mount /dev/loop4p1 /mnt/disk1
mount /dev/loop16p1 /mnt/disk2
mount /dev/loop17p1 /mnt/disk3
```

### Вывод:
```
Монтирование выполнено
```

---

## Пункт 2: Проверка доступности дисков

### Проверка через lsblk:
```bash
lsblk | grep -E "(loop4|loop16|loop17)"
```

### Вывод:
```
loop4        7:4    0     1G  0 loop 
└─loop4p1  259:0    0  1023M  0 part /mnt/disk1
loop16       7:16   0     1G  0 loop 
├─loop16p1 259:1    0   511M  0 part /mnt/disk2
└─loop16p2 259:2    0   511M  0 part 
loop17       7:17   0     1G  0 loop 
└─loop17p1 259:3    0  1022M  0 part /mnt/disk3
```

**Видно:**
- loop4p1 (1023M) смонтирован в /mnt/disk1
- loop16p1 (511M) смонтирован в /mnt/disk2
- loop17p1 (1022M) смонтирован в /mnt/disk3
- loop16p2 (btrfs) не смонтирован

### Проверка через df -h:
```bash
df -h | grep -E "(loop4|loop16|loop17|Filesystem)"
```

### Вывод:
```
Filesystem      Size  Used Avail Use% Mounted on
/dev/loop4p1    989M   24K  922M   1% /mnt/disk1
/dev/loop16p1   447M   35M  413M   8% /mnt/disk2
/dev/loop17p1   988M   24K  921M   1% /mnt/disk3
```

**Результаты:**
- disk1 (ext4): 989M доступно, использовано 24K
- disk2 (xfs): 447M доступно, использовано 35M
- disk3 (ext4): 988M доступно, использовано 24K

## Пункт 3: Автоматическое монтирование через /etc/fstab

### Получение UUID дисков:
```bash
blkid | grep -E "(loop4p1|loop16p1|loop17p1)"
```

### Вывод:
```
/dev/loop17p1: UUID="32fe12b1-ea8d-4017-a01f-492b56cff298" BLOCK_SIZE="4096" TYPE="ext4" PARTLABEL="primary" PARTUUID="7a57f4ad-01c1-4993-b60b-0c72949f9db4"
/dev/loop4p1: UUID="4242abcf-e523-439a-bf6e-66af01a7c5bd" BLOCK_SIZE="4096" TYPE="ext4" PARTUUID="03227aa0-01"
/dev/loop16p1: UUID="461abff5-1922-4568-920e-e7d9cede7aea" BLOCK_SIZE="512" TYPE="xfs" PARTLABEL="primary" PARTUUID="18885fdc-2519-4168-9ad3-461ea43c14e5"
```

### Создание резервной копии fstab:
```bash
cp /etc/fstab /etc/fstab.backup.$(date +%Y%m%d_%H%M%S)
```

### Добавление записей в /etc/fstab:
```bash
cat >> /etc/fstab << 'EOF'

# GigaDevOps virtual disks
UUID=4242abcf-e523-439a-bf6e-66af01a7c5bd /mnt/disk1 ext4 defaults 0 2
UUID=461abff5-1922-4568-920e-e7d9cede7aea /mnt/disk2 xfs defaults 0 2
UUID=32fe12b1-ea8d-4017-a01f-492b56cff298 /mnt/disk3 ext4 defaults 0 2
EOF
```

### Проверка содержимого fstab:
```bash
tail -5 /etc/fstab
```

### Вывод:
```
# GigaDevOps virtual disks
UUID=4242abcf-e523-439a-bf6e-66af01a7c5bd /mnt/disk1 ext4 defaults 0 2
UUID=461abff5-1922-4568-920e-e7d9cede7aea /mnt/disk2 xfs defaults 0 2
UUID=32fe12b1-ea8d-4017-a01f-492b56cff298 /mnt/disk3 ext4 defaults 0 2
```

**Формат записей fstab:**
- Поле 1: UUID диска (устойчивый идентификатор)
- Поле 2: Точка монтирования
- Поле 3: Тип файловой системы
- Поле 4: Опции монтирования (defaults)
- Поле 5: Dump (0 = не создавать дамп)
- Поле 6: Pass (2 = проверять после корневой ФС)

---

## Пункт 4: Проверка автоматического монтирования

### Размонтирование всех дисков:
```bash
umount /mnt/disk1 /mnt/disk2 /mnt/disk3
```

### Вывод:
```
Unmounted
```

### Монтирование через fstab (симуляция загрузки):
```bash
mount -a
df -h | grep "/mnt/disk"
```

### Вывод:
```
/dev/loop4p1    989M   24K  922M   1% /mnt/disk1
/dev/loop16p1   447M   35M  413M   8% /mnt/disk2
/dev/loop17p1   988M   24K  921M   1% /mnt/disk3
```

**Результат:** Команда `mount -a` успешно смонтировала все диски из /etc/fstab. Это подтверждает, что при загрузке системы диски будут автоматически монтироваться.
