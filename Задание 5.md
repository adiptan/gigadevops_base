## Пункт 1: Измерение скорости записи

### Подготовка дисков:
```bash
# Перемонтирование disk1 в обычный режим
umount /mnt/disk1
mount /dev/loop4p1 /mnt/disk1

# Монтирование btrfs раздела для тестирования
mkdir -p /mnt/disk2_btrfs
mount /dev/loop16p2 /mnt/disk2_btrfs
```

### Тест записи ext4 (disk1):
```bash
dd if=/dev/zero of=/mnt/disk1/test_write bs=1M count=100 conv=fdatasync
```

### Вывод:
```
100+0 records in
100+0 records out
104857600 bytes (105 MB, 100 MiB) copied, 0.0970967 s, 1.1 GB/s
```

**Результат ext4:** 1.1 GB/s

**Параметры теста:**
- `if=/dev/zero` - источник данных (нулевые байты)
- `of=...` - выходной файл
- `bs=1M` - размер блока 1 МБ
- `count=100` - количество блоков (100 МБ)
- `conv=fdatasync` - принудительная запись на диск (без кеша)

### Тест записи xfs (disk2):
```bash
dd if=/dev/zero of=/mnt/disk2/test_write bs=1M count=100 conv=fdatasync
```

### Вывод:
```
13+0 records in
12+0 records out
12779520 bytes (13 MB, 12 MiB) copied, 0.0129673 s, 986 MB/s
```

**Результат xfs:** 986 MB/s
**Примечание:** Диск почти заполнен (98%), поэтому записалось только 12 МБ вместо 100.

### Тест записи btrfs (disk2_btrfs):
```bash
dd if=/dev/zero of=/mnt/disk2_btrfs/test_write bs=1M count=100 conv=fdatasync
```

### Вывод:
```
100+0 records in
100+0 records out
104857600 bytes (105 MB, 100 MiB) copied, 0.134431 s, 780 MB/s
```

**Результат btrfs:** 780 MB/s

### Сравнительная таблица скорости записи:

| Файловая система | Скорость записи | Примечание |
|------------------|----------------|------------|
| ext4             | 1.1 GB/s       | Самая быстрая |
| xfs              | 986 MB/s       | Диск почти заполнен |
| btrfs            | 780 MB/s       | COW overhead |

**Анализ:**
- **ext4** показывает лучшую скорость записи благодря оптимизированным алгоритмам
- **xfs** близок к ext4, но ограничен заполненностью диска
- **btrfs** медленнее из-за Copy-on-Write механизма и метаданных

### Вывод:

**Результат btrfs:** 7.7 GB/s

### Сравнительная таблица скорости чтения:

| Файловая система | Скорость чтения | Примечание |
|------------------|----------------|------------|

**Анализ:**

## Пункт 3: Тестирование устойчивости к аварийной перезагрузке

### ⚠️ ВАЖНО: Симуляция НЕ выполнена в учебной среде!

Аварийная перезагрузка может повредить систему, поэтому тест не проводился.

### Теоретический процесс тестирования:

#### Шаг 1: Создание тестовых файлов
```bash
# На ext4
echo "ext4 test data" > /mnt/disk1/crash_test.txt

# На xfs
echo "xfs test data" > /mnt/disk2/crash_test.txt

# На btrfs
echo "btrfs test data" > /mnt/disk2_btrfs/crash_test.txt
```

#### Шаг 2: Симуляция аварийной перезагрузки
```bash
# ОПАСНО! Не выполнять в продакшене!
echo b > /proc/sysrq-trigger  # Немедленная перезагрузка без sync
```

Или отключение питания:
```bash
echo o > /proc/sysrq-trigger  # Немедленное выключение
```

#### Шаг 3: Проверка целостности после перезагрузки

**ext4:**
```bash
# Автоматическая проверка при загрузке
e2fsck -f /dev/loop4p1

# Проверка файлов
cat /mnt/disk1/crash_test.txt
ls -la /mnt/disk1/lost+found/
```

**xfs:**
```bash
# xfs самовосстанавливается при монтировании
mount /dev/loop16p1 /mnt/disk2
xfs_repair -n /dev/loop16p1  # Проверка без изменений

# Проверка файлов
cat /mnt/disk2/crash_test.txt
```

**btrfs:**
```bash
# Проверка и восстановление
btrfs check /dev/loop16p2
btrfs scrub start /mnt/disk2_btrfs
btrfs scrub status /mnt/disk2_btrfs

# Проверка файлов
cat /mnt/disk2_btrfs/crash_test.txt
```

### Ожидаемые результаты:

| ФС | Журналирование | Устойчивость | Восстановление |
|----|---------------|-------------|---------------|
| ext4 | Да (journal) | Высокая | Быстрое (fsck) |
| xfs | Да (log) | Очень высокая | Автоматическое |
| btrfs | Да (COW) | Максимальная | Встроенное (scrub) |

**Анализ механизмов защиты:**

**ext4:**
- Журналирование метаданных и опционально данных
- lost+found для восстановления файлов
- e2fsck для проверки и восстановления

**xfs:**
- Продвинутое журналирование (journaling log)
- Восстановление при монтировании
- xfs_repair для критичных случаев

**btrfs:**
- Copy-on-Write (COW) - запись в новое место
- Checksumming всех данных и метаданных
- Встроенный scrub для проверки целостности
- Snapshots для отката к предыдущим состояниям

## Пункт 4: Отчёт о нагрузке с помощью iostat

### Установка sysstat (содержит iostat):
```bash
apt-get install -y sysstat
```

### Вывод:
```
sysstat is already the newest version (12.6.1-2).
```

### Анализ нагрузки на диски:
```bash
iostat -x 1 2
```

**Параметры:**
- `-x` - расширенная статистика
- `1` - интервал обновления (1 секунда)
- `2` - количество отчётов

### Вывод (фрагмент для наших loop-устройств):

```
Device            r/s     rkB/s   rrqm/s  %rrqm r_await rareq-sz     w/s     wkB/s   wrqm/s  %wrqm w_await wareq-sz
loop4            0.00      0.02     0.00   0.34    0.10    25.01    0.00      1.07     0.00  23.52    1.82   997.91
loop16           0.00      0.06     0.00   0.00    0.07    26.86    0.00      0.97     0.00  61.66    6.60   796.65
loop17           0.00      0.03     0.00   0.41    0.02    28.38    0.00      1.21     0.00  21.03    3.66  1041.09
```

### Расшифровка ключевых метрик:

**Чтение:**
- `r/s` - операций чтения в секунду
- `rkB/s` - КБ/с прочитано
- `r_await` - среднее время ожидания чтения (мс)
- `rareq-sz` - средний размер запроса чтения (КБ)

**Запись:**
- `w/s` - операций записи в секунду
- `wkB/s` - КБ/с записано
- `w_await` - среднее время ожидания записи (мс)
- `wareq-sz` - средний размер запроса записи (КБ)

**Очередь:**
- `aqu-sz` - средний размер очереди запросов
- `%util` - процент времени занятости устройства

### Анализ для наших дисков:

**loop4 (ext4):**
- Запись: 1.07 KB/s
- Средний размер запроса: 997 KB
- Время ожидания записи: 1.82 мс

**loop16 (xfs + btrfs):**
- Запись: 0.97 KB/s
- Средний размер запроса: 796 KB
- Время ожидания записи: 6.60 мс

**loop17 (ext4 с квотами):**
- Запись: 1.21 KB/s
- Средний размер запроса: 1041 KB
- Время ожидания записи: 3.66 мс

### Мониторинг в реальном времени:
```bash
# Обновление каждую секунду
iostat -x 1

# Мониторинг конкретного устройства
iostat -x sda 2

# С временными метками
iostat -x -t 5
```

### Полезные команды для анализа нагрузки:

```bash
# iotop - интерактивный мониторинг I/O процессов
apt-get install -y iotop
iotop

# Детальная статистика одного устройства
iostat -dx /dev/sda 1

# Сохранение статистики в файл
iostat -x 5 > /tmp/iostat.log

# Анализ за определённый период
sar -d 1 10  # Disk statistics каждую секунду, 10 раз
```
